========================
RESEARCH GOAL
========================

I want to compute a DAO-level metric called average participation rate.

The logic is:

1. For each proposal, compute a proposal-level participation rate
2. Then aggregate proposal-level participation rates to the DAO level (e.g. mean, median, weighted mean)

The proposal-level participation rate should be defined as:

participation_rate(proposal) =
    number_of_actual_voters / number_of_eligible_voters

Where:
- number_of_actual_voters = unique addresses that actually cast a vote on Snapshot for that proposal
- number_of_eligible_voters = unique addresses that had positive voting power for that proposal at the relevant Snapshot block

Important:
I do NOT want to confuse:
- "Abstain" = an explicit vote choice by someone who participated
with
- "Did not vote" = an eligible voter who did not submit any vote

So:
- Snapshot votes = actual participation
- Abstain is part of participation
- non-participation must be inferred by comparing actual voters against eligible voters

========================
IMPORTANT METHODOLOGY CORRECTIONS
========================

Please follow these rules exactly:

Rule 1:
Do NOT use the proposal voting period (start block to end block) to define eligibility.
Eligibility should be evaluated at the proposal’s Snapshot block only.

Rule 2:
Do NOT assume token holders = eligible voters.
Snapshot voting power is strategy-driven, not always simple token balance.

Rule 3:
Do NOT assume Etherscan can directly return "all addresses with voting power at a block including delegation".
That is NOT directly available.

Rule 4:
Eligibility must be reconstructed based on the Snapshot voting strategy / strategies configured for the space.

Rule 5:
The main definition of eligibility is:
eligible voter = address with voting power > 0 at the proposal snapshot block

Do NOT use a USD token-value threshold (e.g. $100) in the main definition.
That can only be an optional robustness check later.
