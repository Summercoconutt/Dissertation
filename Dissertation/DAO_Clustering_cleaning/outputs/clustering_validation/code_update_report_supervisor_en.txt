2. PRIMARY SCRIPT: clustering_validation_pipeline.py
--------------------------------------------------------------------------------

Location:
  Dissertation/DAO_Clustering_cleaning/clustering_validation_pipeline.py

Role:
  Research-oriented **validation** pipeline: it does not assume clusters exist; it
  reports diagnostics (distributions, correlations, PCA, K-means over k, hierarchical
  clustering, DBSCAN, column-permutation null benchmark, stability via repeated seeds).

Key configuration (editable at top of file):
  • INPUT_PATH — default Excel feature table (with fallback CSV under DAO_Clustering)
  • FEATURE_COLUMNS — 12 governance / participation / representation features
  • LOG1P_COLUMNS — includes proposal_frequency_per_30d
  • Missing values: rows with NA in selected features are **dropped** (not imputed)
  • Scaling: RobustScaler on the retained feature matrix
  • K range: 2–6; detailed K-means outputs use k with **maximum silhouette** on that sample
  • Two parallel tracks: **all** cleaned DAOs, and **no_outliers** after multivariate
    outlier flagging (z-score / IQR rule)

Main output directory:
  Dissertation/DAO_Clustering_cleaning/outputs/clustering_validation/

Deliverables include CSV metrics, PNG figures, Markdown/text summaries, cleaned datasets,
outlier report, and (at run end) copies of selected tables to the folder root for quick
reference.

5. RECONCILIATION WITH DAO_Clustering/clustering_pipeline.py
--------------------------------------------------------------------------------

Issue:
  clustering_evaluation_k2_to_k6.csv (DAO_Clustering) showed k=2 silhouette ≈ 0.97,
  while clustering_validation (cleaning) showed ≈ 0.23 on the **same underlying
  Excel table** — apparently contradictory.

Investigation (reproducible on the same machine, same dao_feature_table.xlsx):
  The gap is **not** due to loading a different file or a bug in silhouette computation.
  It is driven primarily by one preprocessing difference:

  • clustering_pipeline.py applies **median imputation** for missing values and does
    **not** apply log1p to proposal_frequency_per_30d before RobustScaler.
  • clustering_validation_pipeline.py applies **log1p** to proposal_frequency_per_30d
    and **drops** rows with missing values (only 2 rows dropped in the current data).

Ablation check:
  Adding log1p while keeping median imputation and N=436 reduces silhouette to ≈ 0.23,
  matching the cleaning pipeline. Dropping two rows without log1p leaves silhouette ≈ 0.97.
  Conclusion: **log1p on proposal frequency is the dominant cause** of the metric shift;
  sample size difference is negligible for this comparison.

Implication for the thesis:
  High silhouette in the old pipeline should not be read as “strong natural clusters”
  without acknowledging the raw-scale, heavy-tailed proposal variable. The cleaning
  pipeline’s transform is more conservative for distance-based clustering.

Optional follow-up (not yet committed as default):
  Align clustering_pipeline.py with the same log1p rule (and optionally the same missing-
  value policy) so both scripts become directly comparable for appendix tables.
