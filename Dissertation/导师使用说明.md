# 论文流水线运行说明（供导师 / 服务器环境）

本文说明如何在服务器上获取代码、配置数据路径，并按顺序运行 **DAO 聚类 → 选民聚类 → 行为建模** 相关流水线。默认假设使用 **Linux** 或 **macOS** 终端；文末附 **Windows PowerShell** 环境变量写法。

---

## 1. 代码获取

仓库地址（请将链接发给您的学生确认最新分支名）：

- **GitHub**：`https://github.com/Summercoconutt/Dissertation`
- **建议使用分支**：`thesis-pipeline`（包含完整 Dissertation 代码目录；大体积数据与本地运行产物未纳入 Git，需在服务器上生成）

克隆并进入项目中的论文目录：

```bash
git clone https://github.com/Summercoconutt/Dissertation.git
cd Dissertation
git checkout thesis-pipeline
```

本地工作目录下文记为 **`Dissertation/`**，其中核心子目录包括：

| 路径 | 作用 |
|------|------|
| `DAO_Clustering_cleaning/outputs/.../cluster_assignments.csv` | DAO 层面聚类结果（空间 → DAO cluster），选民聚类与行为建模都会用到 |
| `Voter_Clustering/` | 从 Snapshot 选票构建 master 表，并在 DAO cluster 基础上做选民聚类 |
| `behaviour_modelling/` | 基于聚类后的标签做投票行为序列建模（训练 / 评估） |
| `test_pipeline/` | **小规模冒烟测试**：只选少量 DAO，快速验证环境与代码是否跑通 |

---

## 2. 环境准备

建议使用 **Python 3.10+**（若服务器已有 CUDA，可安装 GPU 版 PyTorch 以加速行为建模）。

### 2.1 虚拟环境（推荐）

```bash
cd Dissertation
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
```

### 2.2 依赖安装

选民聚类与行为建模分开维护了依赖文件，建议一次性安装：

```bash
pip install -U pip
pip install -r Voter_Clustering/requirements.txt
pip install -r behaviour_modelling/requirements.txt
```

可选（选民聚类若启用 HDBSCAN）：

```bash
pip install hdbscan
```

---

## 3. 数据准备（必读）

整条链路依赖两类外部数据：

1. **Snapshot 选票快照**  
   目录结构需符合选民聚类代码的约定：根目录下有若干 `space=*/proposal=*.parquet`（详见 `Voter_Clustering` 内说明）。  
   学生应在交接时交代该目录在服务器的绝对路径。

2. **DAO 聚类归属表**  
   CSV，至少包含列 **`space`** 与 DAO 簇标签（列名可为 **`cluster`** 或 **`dao_cluster`**）。  
   仓库中示例路径（若学生未改动）：

   `Dissertation/DAO_Clustering_cleaning/outputs/cluster_results/no_outliers/cluster_assignments.csv`

若上述 CSV 不在仓库内（体积或隐私原因），请学生单独提供文件路径。

---

## 4. 推荐运行顺序

### 阶段 A：冒烟测试（强烈建议先做）

目的：用 **少量 DAO**、**较短训练**，验证依赖与路径是否正确，无需跑完全量 Snapshot。

前置：仓库内自带 **`Voter_Clustering/data/processed/master_votes_sample.csv`**（样例表决表，体积相对小）。脚本会自动挑选在 `cluster_assignments.csv` 中也有记录的 space，避免无法匹到 DAO cluster。

```bash
cd Dissertation/test_pipeline
python run_demo.py --auto-spaces 2 --epochs 1 --batch-size 16 \
  --window 3 --max-length 48 --max-train-windows 256 --max-valid-windows 128 \
  --eval-max-windows 256
```

生成物默认在 **`test_pipeline/outputs/`**（首次成功运行后可见 `voter_clustering/`、`behaviour_modelling/` 等）。若 Mode A 因样本太小跳过聚类，可暂时降低阈值后再试：

```bash
export VOTER_CLUSTER_MIN_ROWS_MODE_A=10   # 按需要调整
python run_demo.py ...
```

自定义 DAO 列表：将 `demo_spaces.example.txt` 复制为 **`demo_spaces.txt`**，每行写一个与 master 表中一致的 `space` 名称。

---

### 阶段 B：全量选民聚类（从 Snapshot 构建 master）

在配置好 Snapshot 根目录与 DAO CSV 后，进入 `Voter_Clustering`：

**Linux / macOS：**

```bash
export VOTER_SNAPSHOT_ROOT=/path/to/snapshot_votes_data/snapshot_votes_441/spaces
export VOTER_DAO_CLUSTERS_CSV=/path/to/cluster_assignments.csv
cd Dissertation/Voter_Clustering
pip install -r requirements.txt   # 若尚未安装
python run_pipeline.py
```

主要产出包括（路径相对于 `Voter_Clustering/`）：

- `data/processed/master_votes.parquet`、`master_votes_with_dao_cluster.parquet`
- `outputs/voter_clustering/voter_cluster_assignments.csv`
- `outputs/voter_clustering/cluster_summary_statistics.csv`、`figures/` 等

全量扫描前，可在 **`src/voter_clustering/config.py`** 中将 `MAX_PARQUET_FILES` 设为整数（例如 `2000`）做试跑；正式跑全文改为 **`None`**。

---

### 阶段 C：行为建模（接选民聚类输出）

在阶段 B 完成后，master 表与选民簇标签路径默认相对于 `Dissertation/` 如下：

- `Voter_Clustering/data/processed/master_votes_with_dao_cluster.parquet`
- `Voter_Clustering/outputs/voter_clustering/voter_cluster_assignments.csv`

运行：

```bash
cd Dissertation/behaviour_modelling
python run_pipeline.py
```

若文件不在默认位置，可用环境变量覆盖（便于搬到大容量磁盘）：

```bash
export BM_MASTER_VOTES_PARQUET=/path/to/master_votes_with_dao_cluster.parquet
export BM_VOTER_CLUSTER_ASSIGNMENTS_CSV=/path/to/voter_cluster_assignments.csv
export BM_OUTPUT_DIR=/path/to/behaviour_outputs
python run_pipeline.py
```

产出默认在 **`behaviour_modelling/outputs/behaviour_modelling/`**（数据集 CSV、`agent2_artifacts/` 模型与 tokenizer、`eval/` 评估表等）。行为建模默认使用 Hugging Face 预训练模型，**首次运行**需能访问外网下载权重；离线环境请学生预先配置镜像或本地缓存目录（`HF_HOME` / `TRANSFORMERS_CACHE`）。

---

## 5. 资源与耗时提示

- **选民聚类**：CPU 即可；磁盘与内存取决于 Snapshot 规模；`master_votes.parquet` 可达数十 GB 量级时需预留空间。
- **行为建模**：若有 GPU，可将 PyTorch 安装为 CUDA 版本；CPU 亦可运行但较慢。训练脚本支持通过 `--max_train_windows` 等参数缩小试跑规模（见 `behaviour_modelling/train.py` 与 `test_pipeline/run_demo.py` 中的转发参数）。

---

## 6. 常见问题

| 现象 | 可能原因 | 处理建议 |
|------|----------|----------|
| 找不到 `cluster_assignments.csv` | 路径未挂载或未随仓库提供 | 向学生索要绝对路径并设置 `VOTER_DAO_CLUSTERS_CSV` |
| 选民聚类输出为空或大量跳过 | Mode A 下单个 DAO 簇内样本过少 | 调整 `VOTER_CLUSTER_MIN_ROWS_MODE_A` 或增加数据 / 换聚类模式（见 `config.py` 中 `CLUSTERING_MODE`） |
| 行为建模下载模型失败 | 无外网或防火墙 | 配置 Hugging Face 镜像、离线缓存或由学生提供已下载的缓存目录 |
| Git 推送/克隆不含大文件 | 大文件被 `.gitignore` 排除 | 按本文在服务器上重新生成 parquet 与模型，不要依赖仓库内的二进制大文件 |
